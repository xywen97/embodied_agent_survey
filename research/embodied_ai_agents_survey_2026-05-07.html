核心判断

VLA端到端动作模型

OpenVLA、π0/openpi、GR00T、Gemini Robotics、Helix、Redwood 直接把视觉/语言/状态映射到动作。

Planner高层 Agent + 技能库

SayCan、Code as Policies、Inner Monologue、VoxPoser 用 LLM/VLM 做规划、代码生成或 3D grounding。

MCPAgent 接 ROS

RobotMCP/ros-mcp-server、ROSClaw 可把 OpenClaw/Nanobot/Claude/ChatGPT 接到 ROS topic/service。

Product闭源产品加速

Figure、1X、Sanctuary、Tesla、Boston Dynamics、Agility、Unitree 都在把基础模型和具身硬件结合。

最值得跟踪的开源项目

  • LeRobot：机器人学习工具链，统一数据、策略和硬件接口。
  • OpenVLA：开源 VLA，提供 REST API 部署和新 embodiment 微调路径。
  • openpi：π0、π0-FAST、π0.5 开源模型与训练/推理代码。
  • ROS MCP Server：把 MCP agent 接入 ROS/ROS2 的直接工程入口。
  • Octo：基于 Open X-Embodiment 的 generalist robot policy。
  • Code as Policies 与 VoxPoser：LLM 规划/代码生成/3D grounding 的经典路线。

OpenClaw / Nanobot 与硬件的关系

OpenClaw 和 Nanobot 是 agent host/gateway/tool orchestration，不是低层机器人控制器。它们可以作为自然语言入口，通过 MCP server 调用 ROS/SDK；但必须配套权限、白名单、仿真预执行、速度/力矩限制和急停。 
OpenClaw / Nanobot / Claude / ChatGPT
        -> MCP client
        -> ROS-MCP server
        -> rosbridge
        -> ROS topics / services / actions
        -> robot or simulator

推荐实验路线

  1. 用 LeRobot 跑低成本机械臂或 Unitree G1 支持环境，建立数据采集和训练闭环。
  2. 选择 OpenVLA/OFT 或 openpi/π0 做策略微调，优先复现单任务。
  3. 接入 RobotMCP/ros-mcp-server，让 agent 先只读 ROS 状态，再开放白名单动作。
  4. 用 Isaac/MuJoCo/Gazebo 做动作预演，验证碰撞、速度、力矩和工作空间边界。
  5. 展示链路：自然语言指令 -> agent 计划 -> 仿真验证 -> 硬件执行 -> 日志/视频回传。
